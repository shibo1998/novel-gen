# 发布记录

### 2026-07-17

- 自动写整章从逐场景独立生成改为一次生成完整章节，所有场景合同共享同一叙事上下文，减少重复开场、人物口吻漂移和场景间割裂。
- 整章统一执行 2500 字目标与 2300-2700 字接受范围，初稿不合格时最多进行一次带具体问题的定向修订。
- 生成结果通过隐藏边界映射回现有场景，并在完整解析和审校后一次性保存，只创建一个章节内容版本；人工单场景写作保持不变。

### 2026-07-16

- 审校服务不可用或返回错误时，自动写作流程会立即停止并保留失败状态，不再将服务异常当作普通审校意见继续重写。
- 统一写作与审校使用的中文风格规则来源，确保各生成环节应用一致的去 AI 味约束。
- 章节场景规划改为读取上一章时点的 Story Bible 快照，将角色当前状态和关系带入场景约束生成，减少跨章状态漂移。
- 已到回收章节的伏笔作为关键上下文强制保留；普通活跃伏笔仍按预算裁剪，避免到期伏笔被挤出 Writer 提示词。
