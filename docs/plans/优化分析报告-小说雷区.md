你说得对，我之前的分析**混入了太多英文 AI 写作的雷区**——"delve""tapestry""testament"这些词跟中文小说毫无关系。另外你对"对话太有效率"的质疑也有道理，中文网文（尤其玄幻/武侠/悬疑）确实以干净利落的对话为主流。让我重新审视。

# 中文 AI 小说雷区全景图（修正版）

## 先回应你的质疑：对话"干净利落"vs AI 式"高效率"

你说的没错——**干净利落不是问题**。问题在于：

> **所有角色都以「同一种方式」干净利落。**

《诡秘之主》的克莱恩说话谨慎迂回，奥黛丽说话优雅但天真，阿尔杰说话粗粝直接——每种都很高效，但你闭着眼也能分辨是谁在说话。AI 的问题是：林远、苏清霜、玄明三个人物的对话**句式结构、信息完整度、礼貌程度完全一致**——这才是"AI 味"的来源，不是效率本身。

所以纠正后的定义：

- ✅ **角色辨识度缺失**——每个角色都应该有独特的"说话指纹"（句式长度、主动/被动、是否用反问、是否留半句）
- ❌ ~~对话太有效率~~（不够精确，删掉）

## 🔴 致命级：专属于中文 AI 小说的"一眼 AI"

### 1. "只见/只听得/只感觉"——中文 AI 的万能描写引入词

英文 AI 没有这个问题，这是**中文 AI 独有**的最严重指纹。

**实际表现**：

> 只见一道黑影从屋檐掠过。只听得风中传来一声冷笑。林远只感觉一股寒意从背后袭来。

人类作家会用不同方式引入描写——有时直接描写，有时用角色动作过渡。AI 几乎每段新描写都用"只见"开头，三章下来读者就审美疲劳了。

**防御方案**：Writer Agent Prompt 中增加硬性规则——

```
禁止使用"只见/只听得/只感觉/只觉"作为描写引入语。
用具体动作或感官细节替代。
❌ "只见一道剑光闪过"
✅ "剑光破空。林远侧身，剑锋擦着耳廓过去。"
```

### 2. "眼中闪过"+"心头一颤"——情绪描写的万能公式

AI 不会让角色**做**什么来表达情绪，只会写"眼中闪过"或"心头一震"。

**实际表现**：

> 林远眼中闪过一丝复杂之色。
> 苏清霜心头一颤。
> 玄明眼中闪过一抹杀意。

这些都是**公式化的增量**——"一丝 + 形容词 + 的 + 名词"结构。人类作家同样会描写微表情和身体反应，但不会只用这一个句式。

**防御方案**：在 `writer_event.j2` 中增加——

```
禁止使用"眼中闪过一丝X"或"心头一颤/一震/一紧"作为情绪描写的默认句式。
将 internal_state 转化为可观察的行为：
❌ "林远眼中闪过一丝愤怒"
✅ "林远的手指停在茶杯边缘，不动了。"
```

### 3. "一股...涌上心头/心中/心底"——AI 不会写"感觉"，只会命名"感觉"

**实际表现**：

> 一股暖流涌上心头。一股不祥的预感涌上心底。一股复杂的情绪在他心中翻涌。

问题跟第 2 点一样——AI 用"命名情绪"代替"展示情绪"。

### 4. 中文破折号"——"泛滥

中文 AI 同样爱用破折号，而且中文破折号（——）比英文 em dash 更占视觉空间，泛滥后极其扎眼。

**防御方案**：审校 Agent 检测"——"密度。每 1000 字超过 1 个即警告。

### 5. "不是...而是..." / "与其说...不如说..."——中文版 Paradiastole

**实际表现**：

> 这不是一场普通的战斗，而是一场灵魂的试炼。
> 与其说他是剑客，不如说他是一个被剑选中的祭品。

跟英文版一样的问题——用公式制造虚假深度。

## 🟠 显眼级：写作者和编辑能识别

### 6. 对话标签三件套："淡淡道""冷冷道""沉声道"

中文 AI 的对话标签极其单调。90% 的非"说/道"标签集中在三个词：

| AI 最爱                    | 占比 | 正常人类写作占比 |
| -------------------------- | ---- | ---------------- |
| 淡淡道                     | ~30% | ~5%              |
| 冷冷道                     | ~20% | ~3%              |
| 沉声道                     | ~15% | ~5%              |
| 其他（缓缓道/轻声道/叹道） | ~25% | —                |
| 动作节拍替代               | ~10% | ~60%             |

**防御方案**：Phase 3.5 `character_dialogue.j2` 中增加——

```
对话标签使用规则：
1. "说/道"类标签不超过全部对话的 15%
2. 禁止连续三句对话使用标签
3. 60% 的对话用动作节拍替代标签
4. 每个角色有专属的动作节拍库（林远=摩挲东西/沉默/看窗外，苏清霜=整理衣袖/挑眉/不直视对方）
```

### 7. 中文 AI 高频词表（专属版）

| 类别     | AI 高频词                            | 建议替代/处理              |
| -------- | ------------------------------------ | -------------------------- |
| 过渡     | 然而、与此同时、就在此时、当下、当即 | 直接删掉或用动作过渡       |
| 模糊     | 仿佛、似乎、不知为何、莫名           | 减少 70%，用确切的描写替代 |
| 慢动作   | 缓缓、徐徐、渐渐、微微               | 减少 80%，用具体动作替代   |
| 描写引入 | 只见、只听得、只感觉、映入眼帘       | **禁用**，用直接描写替代   |
| 情绪     | 心中暗道、暗想、心头一颤、眼中闪过   | **禁用**，用行为展示       |
| 气息     | 一股...的气息、...扑面而来           | 减少 90%                   |
| 结尾     | 就这样、从此、在这一刻               | **禁用**于段落/章节结尾    |

### 8. 成语堆砌代替描写

AI 喜欢用成语来"偷懒"——用四个字概括本该展开描写的场景。

**实际表现**：

> 两人你来我往，打得天昏地暗、日月无光。

这是**偷懒**。人类作家会写三到五个具体的动作交换，让读者感受到战斗的节奏。AI 用一个成语就把整场战斗跳过了。

**防御方案**：审校 Agent 增加"成语密度检测"。每 500 字成语超过 3 个 → 警告。战斗/动作场景中，成语占描写比例超过 20% → 要求重写。

## 🟡 微妙级：全书读完才能察觉

### 9. 情节节拍过于均匀（通用问题，不分语言）

中文网文也有这个问题。AI 生成的每章：开场 → 小冲突 → 推进 → 悬念 → 收尾。每章 2200 字，不多不少。而《诡秘之主》有的章 1500 字戛然而止，有的章 5000 字酣畅淋漓。

**防御方案**：在 Phase 2 大纲生成时，显式为每章标注"节奏标签"和"字数范围"，确保方差存在。

### 10. "太干净"——没有毛刺的完美散文

AI 的每一段都有头有尾，每个比喻都贴切，每个场景都"完整"。但实际上好小说有毛刺——一个突兀但绝妙的比喻、一处故意不完整的描写、一段让读者感到不适的留白。

**防御方案**：Phase 5 审校 Agent 增加"毛刺率"指标。如果一章中所有句子长度都在 15-25 字、所有段落都是 3-5 句话、所有场景都有完整收尾 → 标记"过于平滑"，随机要求 Writer Agent 打破 1-2 处模式。

## 最重要的洞见（中文版）

上面所有雷区，本质上是同一个问题：

> **AI 中文写作是在模仿"已经被写出来的中文"，而不是在写"还没有人写过的东西"。**

"只见""眼中闪过""心头一颤""淡淡道"——这些在训练语料（网文、古风小说、传统文学）中确实是高频模式。AI 忠实地学习了这些模式，但没学会**什么时候不该用它们**。

真正的好小说从来不是"最多人使用的中文"——而是"只有这个作者才会这样写的中文"。

你的架构中最关键的不是某个 Prompt 或 Agent，而是 **Phase 7 的 Human-in-the-Loop**。因为只有人的品味能判断"这段虽然符合所有规则，但就是没有灵魂"。AI 可以帮你避开雷区，但没办法帮你写出只有你才能写出的东西。
