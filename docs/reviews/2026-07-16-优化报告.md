## 你问"怎么让小说内容更好"，答案在三个层面

### 层面一：你的系统本身在"内容质量"上的硬伤

审查报告挖出的东西非常诚实。你的系统有四个直接影响小说内容质量的结构性问题：

**1. 语义召回从未落地（问题 3）**

`vector_store.py` 根本不存在。Qdrant 依赖装了但一行代码没写。实际召回是靠「最近 500 条记录按章节倒序 + 中文 bigram 子串碰运气」。一部 90 章的小说写到第 50 章，第 3 章的设定连候选池都进不了——不是排不上分，是根本没进评分池。`context_builder` 给 Writer 喂的上下文里缺什么，Writer 写的正文就缺什么。

**2. 规划阶段是记忆盲区（问题 1）**

ChapterAgent 生成约束卡时只看大纲和角色档案，完全看不到：前面埋了什么伏笔、前情实际发生了什么、角色情绪和关系现在是什么状态。约束卡是 Writer 的唯一指挥棒——指挥棒里没有"该收伏笔了"，Writer 就不会主动收。一致性防线在规划和写作之间存在断层。

**3. 伏笔永远不会回收（问题 13）**

`resolve_foreshadowing()` 全仓零调用。伏笔埋了之后永远 `pending`。更糟的是——`context_builder` 每章都往 prompt 注入"所有活跃伏笔"，第 1 章的伏笔一直活到第 90 章，配合 `max_entries:10 + truncate_oldest`，旧伏笔永久挤占配额，真正相关的被截断。

**4. 两套质量评审互相不沟通（问题 6）**

Reviewer（同步场景级审校）和 QualityWorkflow（异步章级打分）对同一份内容各自评审、互不共享结论。Reviewer 判了 pass，Evaluator 可能又打低分。没有统一的"这一章到底算不算过"的状态。

### 层面二：你的执行计划已经有了——按什么顺序做

审查报告里 Phase 1-12 已经拆好了。关键执行顺序：

| 优先级   | 做什么                                                    | 对内容质量的直接影响                                                |
| -------- | --------------------------------------------------------- | ------------------------------------------------------------------- |
| **第一** | Phase 1：补语义召回链路（pgvector）                       | 第 50 章后旧情节能找回。这是地基。                                  |
| **第二** | Phase 2 + Phase 11：规划-记忆断层 + 伏笔回收闭环          | 约束卡里有"该收的伏笔"，Writer 写的时候知道该呼应的前情。伏笔能收。 |
| **第三** | Phase 3：Bible 召回增强（别名匹配 + 接线 `apply_change`） | 角色状态随章节变化被持久化，不永远是初始档案。                      |
| **第四** | Phase 5：质量门统一                                       | 一章写完后只有一个结论，不重复烧钱重写。                            |

其余 Phase（安全、事务、成本熔断）是该做的，但不直接改善内容质量。Phase 6 的 DHO 自动化是锦上添花——大纲能动态调整，适配"写作时角色发生意料之外的变化"。

### 层面三：回到你的文本本身

你写的这段开篇，和我读完你的审查报告之后的认识是**同构的**。

你在写的时候，陆衡的每一个反应都经过了你的大脑加工——"他感受到了什么"→ 你帮他翻译成文字→读者通过文字接收。但你跳过了中间的"触发→感受"环节，直接给了"感受→定义"。"涨租后差额每月缺口将在饮食支出里自行消化"——这不是陆衡在想，是你在替陆衡给房东的涨价做一个会计审计。

你的系统在做一模一样的事。ChapterAgent 生成约束卡 → Writer 写正文，中间跳过了"查该收的伏笔→查前情实际发生了什么→查剧情线索现在的状态"。约束卡直接给 Writer，但约束卡里没有"前情提要"。

**你的文本问题、你的系统问题、你之前说我的改写"依然不行"——三者是同一个病根的不同症状。**

信息到达接收者之前，跳过了中转站。在你的系统里，中转站是语义召回和伏笔调度。在你的文本里，中转站是读者跟着角色一起接收信息的那个体验链。
