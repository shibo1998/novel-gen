## 一、审计中的事实错误

### 错误 1：「熔断器未接入」

**真相**：`client.py` 第 90 行 `_circuit_breaker = CircuitBreaker()`，模块级单例。第 110-131 行每次流式调用都先 `can_attempt()`，失败调 `record_failure()`，成功调 `record_success()`。

**熔断器是全量接入的。** 你的审计说反了。

### 错误 2：「ContextBuilder 未被调用」

**真相**：`api/writing.py` 第 72-77 行 `_build_writing_context()` 调用 `ContextBuilder.build_context_with_budget()`，产出「上下文报告」，然后通过 `builder.enrich_constraint(constraint, context)` 将 Bible、前情摘要、伏笔、记忆、情节线索注入到 `SceneConstraint` 对象中。

WriterAgent 的 `_build_prompt`（writer.py 第 58-68 行）消费这些字段：

```python
template.render(
    constraint_card=constraint,
    injected_bible=constraint.injected_bible,              # ✅ 全量注入
    injected_previous=constraint.injected_previous,         # ✅ 全量注入
    injected_foreshadowings=constraint.injected_foreshadowings, # ✅ 全量注入
    injected_memories=constraint.injected_memories,         # ✅ 全量注入
    injected_plot_threads=constraint.injected_plot_threads, # ✅ 全量注入
)
```

调用链为：`API → ContextBuilder → SceneConstraint 富化 → WriterAgent._build_prompt`。

**ContextBuilder 已全量接入写作主流程。** 你的审计说反了。

### 错误 3：「权限校验缺失」

**真相**：`api/writing.py` 第 89-99 行 `_get_owned_scene` 通过 JOIN `project_users` 表验证 `scene_id` + `current_user_id` 的归属。第 167 行和第 306 行两个端点都调用了这个函数。

**权限校验存在且完整。** 你的审计说反了。

### 错误 4：「LLM 可靠性与成本控制未接入」

**真相——三层接入状态**：

| 层级                | 位置                                  | 接入状态                                                  |
| ------------------- | ------------------------------------- | --------------------------------------------------------- |
| L1: 熔断器          | `client.py:90`                        | ✅ 全量                                                   |
| L2: 流式重试        | `client.py:22-33`                     | ✅ 全量（第一 token 前重试）                              |
